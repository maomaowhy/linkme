# 项目状态总览

## 1. 项目目标
`Link Me` 是一个面向局域网场景的多端互传项目，目标是支持：
- 桌面端打包为 Windows `.exe` / macOS `.dmg`
- 移动端打包为 Android `.apk`
- 设备通过局域网建立连接后，在手机与电脑之间、手机与手机之间进行文本与文件互传
- 支持多设备会话、连接审批、断开连接、批量文件与文件夹传输、逐文件结果展示

当前项目优先级是“先做出可用成品，再逐步补原生能力与细节优化”。

## 2. 当前架构状态
当前仓库采用 `pnpm workspace` 单仓结构：
- `apps/desktop`：Electron 桌面端 Host
- `apps/mobile`：`uni-app Vue3 + Vite` 移动端
- `packages/shared`：共享协议、二维码 payload、传输 manifest、工具函数

当前通信模型：
- 控制通道：`WebSocket`
- 数据通道：`HTTP` 直接上传
- 连接方式：桌面端显示二维码，移动端扫码或粘贴 payload 建立连接

## 5. 当前限制与风险
- 当前最完整可用的移动端互传体验仍是 `H5` 调试形态
- `app-plus` 当前已可编译，但还缺原生文件与扫码能力，因此暂不能作为“完整可交付 APK 功能版”
- 桌面端 `electron-builder` 目录打包已通过配置验证，但若拉取 Electron 发行包时网络异常，打包会因外部下载失败中断
- 当前不支持断点续传、暂停/恢复、云端暂存、公网连接

## 7. 当前推荐使用方式
- 若要验证核心互传功能，优先使用：桌面端 Electron + 移动端 H5
- 若要验证 Android 壳构建链，使用：`uni-app app-plus` 构建后通过 HBuilderX 继续打包
- 若要验证未来 APK 能力边界，先关注连接壳、页面结构、原生能力接入点是否合理
