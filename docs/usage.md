# 使用说明

> 关联文档：`docs/project-status.md`、`docs/deployment-verification.md`

## H5 / 浏览器调试形态
### 手机推送到电脑
1. 打开桌面端并启动服务
2. 手机端点击“扫码”后，可选择“扫一扫”或“从相册选图”，识别成功后自动填充二维码内容
3. 在桌面端允许连接
4. 手机端选择文件或文件夹后点击“开始推送”
5. 桌面端在对应会话里点击“选择位置并接收”
6. 选择本次批次的保存目录后开始连续推送
7. 若某个文件失败，后面的文件仍继续推送
8. 推送完成后可在两端查看每个文件的结果
9. 若有失败项，可点击“重试失败项”仅重传失败文件

## app-plus 当前状态
- 已可编译成 `uni-app app-plus` 构建产物
- 可手动粘贴二维码内容并验证连接入口
- `H5` 已支持扫一扫与相册选图识别二维码；`app-plus` 已支持 `uni.scanCode` 与相册选图二维码识别，原生文件选择/文件夹选择仍待接入

## 桌面端接收策略
- 每个批次单独选择保存目录
- 可直接拒绝某次推送任务
- 同名文件自动重命名，不覆盖原文件
- 可对某个手机会话主动断开连接

## 多设备
- 同一桌面 Host 可同时允许多个手机连接
- 每个手机独立一个会话，任务互不影响
- 某个手机断开不会影响其他手机的传输

## 当前未完成能力
- 手机作为接收端 / Host
- 手机到手机直连
- 桌面端向手机端推送文件
- app-plus 原生文件选择与扫码适配
