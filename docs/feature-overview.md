# 功能说明

> 关联文档：`docs/project-status.md`、`docs/deployment-verification.md`

## 已实现目标
- 桌面 Host 基础架构
- 移动端 Client 基础架构
- 移动端已迁移到官方 `uni-app Vue3 + Vite` CLI 结构
- 共享协议与任务模型
- 审批式连接模型
- 多文件/文件夹 manifest 建模
- 桌面 Host 的 HTTP 上传端点
- 移动端上传请求构造与发送 helper
- 批次任务与文件级结果状态
- 单文件失败后继续后续推送
- 仅重试失败项
- 电脑端手动选择接收保存目录
- 电脑端可拒绝某次推送任务
- 文件冲突自动重命名保存（`name (1).ext`）
- 桌面端批次结果面板与失败项重试按钮
- 移动端 `h5` 与 `app-plus` 构建通过
- 桌面端轮询刷新待审批设备与会话状态

## 当前限制
- 手机 Host 为 Phase 2 增量能力，当前未实现本地服务能力
- 首期不支持断点续传
- 首期不支持暂停/继续，仅支持失败后重试或断开连接
- 首期仅支持局域网 Wi-Fi
- 桌面端到手机端的文件推送尚未完成
- `H5` 当前已支持扫一扫、选图二维码识别与文件选择；`app-plus` 已支持扫码与选图识别，但仍缺原生文件/文件夹选择，因此完整文件互传能力当前仍以 H5 形态最完整
